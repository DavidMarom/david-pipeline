# Task: David Pipeline Landing Page

## Problem
David Pipeline is an open source template pipeline for developers. It currently has no front page — visitors landing on the project have nothing that communicates what it is, why it exists, or how to get started. First impressions matter for open source adoption.

## Goal
Create an attractive, developer-focused landing page that makes David Pipeline instantly understandable and compelling to developers who discover it.

## Requirements
- Display the project name "David Pipeline" prominently as the hero headline
- Communicate what the project is (open source template pipeline for developers)
- Include a short tagline or description that speaks to developers
- Include a call-to-action (e.g. "Get Started" or "View on GitHub")
- Highlight key features or benefits in a scannable format (3–4 items)
- Visually attractive and modern — must feel like a professional OSS project
- Fully responsive (mobile, tablet, desktop)
- Accessible (semantic HTML, sufficient contrast)

## Constraints
- Built with Next.js (this project's stack) and CSS Modules
- No external UI libraries — use custom styles
- Must work as the root route (`/`)

## Out of scope
- Authentication or user accounts
- Actual pipeline functionality or dashboard
- Blog, changelog, or docs pages
- Analytics integration
